# PRD — SignBridge

> 청각 장애인을 위한 영상 기반 의사소통 보조 플랫폼

---

## 1. Product Overview

SignBridge는 청각 장애인과 비장애인이 **영상 통화를 통해 원활하게 의사소통할 수 있도록 지원하는 웹 기반 커뮤니케이션 플랫폼**이다.

서비스는 음성 중심 커뮤니케이션의 한계를 해결하기 위해 다음 기능을 제공한다.

- 음성 → 실시간 자막 변환 (STT)
- 텍스트 → 음성 변환 (TTS)
- 영상 기반 대화 (WebRTC)
- 빠른 응답 버튼 (Quick Reply)
- 수화 → 텍스트 변환 (확장 기능, 지연 허용)

본 서비스는 **완전한 실시간 수화 통역이 아니라 대화 성공률을 높이는 보조 도구**를 목표로 한다.

---

## 2. Problem Statement

청각 장애인은 음성 중심 서비스 환경에서 의사소통에 어려움을 겪는다.

**대표적인 상황**

- 병원 상담
- 관공서 문의
- 고객센터 상담
- 화상 상담
- 온라인 회의

**현재 존재하는 문제**

1. 음성 정보를 실시간으로 이해하기 어렵다
2. 자신의 의사를 전달하기 위해 반복적인 입력이 필요하다
3. 전화 기반 상담 서비스 이용이 어렵다

결과적으로 **의사소통 장벽이 발생**한다.

---

## 3. Product Goal

서비스의 핵심 목표는 다음과 같다.

1. 청각 장애인과 비장애인의 **대화 흐름을 유지**한다
2. 음성 중심 커뮤니케이션을 **텍스트 기반으로 확장**한다
3. 영상 기반 인터페이스를 통해 **표정 및 제스처를 활용한 대화 환경**을 제공한다

---

## 4. Target Users

### Primary Users — 청각 장애인

- 음성 기반 커뮤니케이션 사용이 어려움
- 자막 기반 정보 전달 선호
- 수화 사용 가능

### Secondary Users — 비장애인

- 병원 직원
- 상담 직원
- 일반 사용자

---

## 5. Core Use Case

### Scenario: 병원 상담 상황

1. 환자와 상담 직원이 영상 통화를 시작한다
2. 직원이 말하면 음성이 자동으로 자막으로 변환된다
3. 청각 장애인은 자막을 읽고 이해한다
4. 청각 장애인은 텍스트 입력 또는 수화로 응답한다
5. 응답이 음성 또는 텍스트로 상대방에게 전달된다

---

## 6. Key Features

### 6.1 영상 통화

사용자는 웹앱에서 영상 통화를 시작할 수 있다.

| 항목 | 내용 |
|------|------|
| 기능 | 1:1 영상 통화, 카메라/마이크 제어, 연결 상태 표시 |
| 기술 | WebRTC |

### 6.2 음성 → 자막 변환 (STT)

비장애인이 말하면 음성을 텍스트로 변환한다.

| 항목 | 내용 |
|------|------|
| 기능 | 실시간 자막 표시, 자막 자동 스크롤, 자막 강조 UI |
| 기술 | Speech-to-Text API |

### 6.3 텍스트 → 음성 변환 (TTS)

청각 장애인은 텍스트로 응답할 수 있다.

| 항목 | 내용 |
|------|------|
| 기능 | 텍스트 입력, 음성 변환, 상대방에게 음성 전달 |
| 기술 | Text-to-Speech API |

### 6.4 빠른 응답 기능 (Quick Reply)

자주 사용하는 문장을 버튼 형태로 제공한다.

| 버튼 예시 |
|-----------|
| 다시 말씀해 주세요 |
| 천천히 말씀해 주세요 |
| 잠시만 기다려 주세요 |
| 이해했습니다 |

목표: 입력 시간을 줄이고 대화 흐름을 유지한다.

### 6.5 수화 → 텍스트 변환 (확장 기능)

사용자가 카메라 앞에서 수화를 하면 텍스트로 변환한다.

| 항목 | 내용 |
|------|------|
| 지연 허용 여부 | 약간의 지연 허용 |
| MVP 범위 | 제한된 수화 단어 인식 (5~10개) |
| 인식 예시 | 네, 아니요, 감사합니다, 도움, 잠시만요 |

## 7. User Flow

### 통화 시작

```
사용자 접속
  → 통화방 생성 (닉네임 입력)
  → 초대 코드 공유
  → 상대방 입장
  → 영상 통화 연결
```

### 대화 흐름

```
비장애인 음성
  → STT
  → 자막 표시
  → 청각 장애인 확인

청각 장애인 응답
  → 텍스트 입력 또는 수화 입력
  → TTS 또는 텍스트 전달
  → 상대방 확인
```

## 8. System Architecture (Overview)

```
[Next.js Frontend]
    ├─ WebRTC Peer Connection
    ├─ WebSocket Client
    ├─ Caption UI
    └─ Text Input / TTS Playback

         │ HTTPS / WSS

[NestJS Backend]
    ├─ Room Service
    ├─ WebRTC Signaling Gateway
    ├─ Caption (STT) Service
    ├─ TTS Service
    └─ AI Integration Layer

         │
         ├─ PostgreSQL
         ├─ STT API
         └─ TTS API
```
